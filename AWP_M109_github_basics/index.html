Git nedir? Neden kullanilir? Nasil kurulur?

Git, acik kaynak kodlu bir versiyon kontrol sistemidir. Neden kullanildigini su sekilde ozetleyebiliriz:

  • Yazmis oldugumuz projenin gelisim sureclerini izleyebiliriz.
  • Projede yapılan degisikliklerin nerede ve ne zaman yapildigini tespit edebiliriz.
  • Projede bir hata meydana geldiginde son stabil calisan versiyonuna geri donebiliriz.
  • Takim olarak calisilan projelerde hangi takim uyesinin hangi degisiklikleri yaptigini tespit edebiliriz.

Git kurulumu için sitesinden kurulum dosyasini indirip standart bir program kurulumu gibi bilgisayariniza kurabilirsiniz.

Github nedir? Temel calisma akisi (Github flow) nasildir?

İlk olarak biraz Git'ten bahsedelim. Git'te 3 bolum bulunur. Bunlar working directory, staging area ve local repo'dur. Proje dosyalarimiz uzerinde bazi degisiklikler yaptiktan sonra dosyalarimizi ilk olarak git add komutu ile staging area alanina kopyalariz. Burada yaptigimiz degisiklikleri inceledikten sonra herhangi bir problem yoksa artik dosyalarimizi git commit komutu ile local repo'ya kopyalariz. Bu asamadan sonra ise git push komutu ile local'deki repomuzu GitHup sunucularına gondeririz. Yine GitHub'ta bulunan repository'leride local bilgisayarimiza pull edebiliriz.

Temel Git Komutlari ve Aciklamalari

Git Komutu Aciklamasi
git config --global user.name = "userName" Bu komut ile kullanici adimizi belirtiyoruz.
git config --global email= "email" Bu komut ile mail adresimizi belirtiyoruz.
git init Calişma klasorumuzun icerisinde local repomuzu olusturur. ".git" dosyasi eklenir. Bu dosya gorunmezse gizli klasorleri goster seceneğini aktif etmemiz gerekiyor.
git status Calisma dosyalari ile Staging area alanindaki dosyalar arasindaki farklari gosterir.
git add Tum dosyalari calisma alanindan staging area alanina kopyalar.
git commit -m "Aciklama Bilgisi" Staging area'da bulunan dosyalari git'e kopyalar.
git push Local repomuzu GitHub hesabimiza push eder.
git show Son yaptigimiz commit'i gosterir.
git log Yaptigimiz tum commit'leri gosterir.
git branch branchName Yeni bir branch olusturur.

Bilgisayarda yeni bir repository olusturma ve Github'a yukleme adimlari

  1. git init
  2. git add .
  3. git commit -m "Aciklama"
  4. GitHub da bos bir repository olusturuyoruz
  5. git remote add "GitHub Repository Name" "GitHub Repository Url"
  6. git push -u "GitHub Repository Name" master
  7. GitHub kullanici adi ve parola girilir
  8. Bu asamadan sonra bilgisayardaki repo GitHub'a yuklenir

Github'da bulunan bir repository'i bilgisayara kaydetme ve yeni dosyalarla Github'a tekrar yukleme adimlari

  1. git clone "clone linki"
  2. git add .
  3. git commit -m "Aciklama"
  4. git push
